# 软件杯 A4 智能仓储全局沙盘 Vibe Coding PRD

## 1. 业务流程与核心约束
赛题本质为2个起点到4个终点的物流搬运系统。所有代码生成需严格遵守以下物理法则与计分约束，踩中一票否决项直接0分。

* **硬件与容量约束**：传送带缓冲区最大6件；AGV单次最大载重 1m³；交接区总容积 1m³；货架机器人最大举升 1m³（须同体积）。
* **一票否决项**：发生物理碰撞、违反体积/容量约束、库存映射不一致。
* **核心目标**：压榨搬运消耗时间（T/s），提升公式得分 $Sn=D*3000/(T+TO)$。

## 2. 工程架构设定
* **后端（核心控制）**：纯 C 语言。独立运行五大核心算法模块，提供 WebSocket 服务。
* **前端（可视化与交互）**：混合架构。外围采用 React (Vite) 搭建面板与日志，沙盘动画必须封装为纯原生 JS 的 `CanvasRenderer` 类，通过 `useRef` 接收后端帧数据高频重绘，确保性能。

## 3. 五大核心算法模块设计

按照赛道系统化设计，后端 C 语言逻辑严格拆分为以下 5 个模块逐步实现：

### 模块一：全局状态管理 (防0分底座)
* **职责**：作为所有算法的底层数据库，记录地图、AGV、货架盘位的实时状态。
* **实现标准**：采用纯内存结构体（Map, AGV_List, Shelf_Matrix）。必须在任何物理动作前，设置严格的防超载、防容量超限的强制校验宏（Assert）。

### 模块二：单车底层寻路
* **职责**：确定单个 AGV 如何从 A 点移动到 B 点，提供基础路径。
* **实现标准（冲刺方案）**：实现带有曼哈顿启发函数的时空间 A* 算法（Time-Space A*）。在节点搜索时引入时间维度 `(x, y, t)`，将动态障碍物（其他车的预约位置）视为不可通行节点。

### 模块三：多车调度与防撞
* **职责**：解决十字路口和窄通道的堵车与会车问题，绝对避免碰撞。
* **实现标准（终极冲刺方案）**：实现 CBS（基于冲突的搜索）算法作为顶层调度。检测底层 A* 规划出的各车路径是否存在“顶点冲突”（同一秒在同一格）或“边冲突”（相向交换）。若存在，则生成时空约束，强制单车重新进行 A* 寻路。

### 模块四：订单贪心拼单
* **职责**：解决传送带缓冲区运力闲置问题，凑满 1m³ 发车。
* **实现标准（冲刺方案）**：实现基于 0-1 背包的动态规划（DP）算法。不仅以拼满 1m³ 为目标，必须将“货物目标货架的距离”作为权重加入计算，优先打包发往同区域的货物。

### 模块五：货架碎片整理
* **职责**：利用业务间隙释放交接区连续空间，提升上架效率。
* **实现标准（冲刺方案）**：实现启发式盘位整理回调函数。当交接区与货架机器人空闲时，扫描三维数组 `Shelf_Matrix`，将浅层的高频区 0.25m³ 货物合并至深层冷门区。
